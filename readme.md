## 3. 多线程与CAS
CAS (Compare-And-Swap) 是一种实现无锁并发算法的核心技术。
它包含三个操作数：内存位置、预期原值、新值。执行时，当且仅当内存位置的值等于预期原值时，将该内存位置更新为新值，否则不进行任何操作。

相比于使用互斥锁（Mutex），CAS 的优点包括：
- **避免死锁**：线程不会因持有锁而被阻塞。
- **性能更好**：在低冲突场景下，减少了线程上下文切换的开销。

项目中 `multi_thread/src/casTest.cpp` 提供了两个使用 CAS 的经典示例：
1. **LockFreeStack**: 使用 CAS 实现线程安全的压栈操作。
2. **ObjectHeader**: 使用 CAS 更新对象头中的标记位。
